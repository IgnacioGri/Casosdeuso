DESCRIPCIÓN

Este caso de uso permite a los analistas de sistemas generar documentación técnica profesional utilizando múltiples modelos de inteligencia artificial. El sistema guía al usuario a través de un formulario estructurado de 9 pasos, aplicando automáticamente las normas ING y generando documentos en formato DOCX con el estilo corporativo adecuado. Incluye funcionalidades de asistencia inteligente para mejorar la calidad del contenido y garantizar el cumplimiento de estándares técnicos.

FLUJO PRINCIPAL DE EVENTOS

  1. Buscar datos de casos de uso existentes
    1. El sistema presenta filtros de búsqueda:
      1. Código del caso de uso
      2. Nombre del caso de uso
      3. Cliente
      4. Proyecto
      5. Tipo de caso (entidad/api/servicio)
      6. Modelo de IA utilizado
      7. Estado del documento
      8. Fecha de creación
    2. El sistema muestra resultados con las siguientes columnas:
      1. Código del caso
      2. Nombre completo
      3. Cliente
      4. Proyecto
      5. Tipo
      6. Modelo IA
      7. Estado
      8. Fecha de creación
      9. Acciones (Editar/Eliminar/Exportar)
  2. Crear un nuevo caso de uso
    1. El sistema presenta los siguientes campos de la entidad:
      1. codigoCasoUso (texto, 20 caracteres, obligatorio)
      2. nombreCaso (texto, 100 caracteres, obligatorio)
      3. clienteNombre (texto, 50 caracteres, obligatorio)
      4. proyectoNombre (texto, 100 caracteres, obligatorio)
      5. tipoCaso (texto, 20 caracteres, obligatorio)
      6. modeloIA (texto, 20 caracteres, obligatorio)
      7. descripcion (texto, 500 caracteres, obligatorio)
      8. filtrosBusqueda (texto, 1000 caracteres, opcional)
      9. columnasResultado (texto, 1000 caracteres, opcional)
      10. camposEntidad (texto, 2000 caracteres, opcional)
      11. reglasNegocio (texto, 2000 caracteres, opcional)
      12. requerimientosEspeciales (texto, 2000 caracteres, opcional)
      13. descripcionWireframes (texto, 2000 caracteres, opcional)
      14. flujosAlternativos (texto, 2000 caracteres, opcional)
      15. contenidoGenerado (texto, 10000 caracteres, opcional)
      16. fechaCreacion (fecha, obligatorio)
      17. usuarioCreacion (texto, 50 caracteres, obligatorio)
      18. fechaModificacion (fecha, opcional)
      19. usuarioModificacion (texto, 50 caracteres, opcional)
    2. Al crear el registro se almacena automáticamente la fecha y usuario de alta

FLUJOS ALTERNATIVOS

  1. Modificar o actualizar un caso de uso existente
    1. El sistema permite editar todos los campos de la entidad excepto el código
    2. El sistema muestra el identificador único del registro
    3. El sistema conserva y muestra la fecha y usuario de alta original
    4. Al modificar se registra automáticamente la fecha y usuario de modificación
  2. Eliminar un caso de uso
    1. El sistema verifica que no tenga documentos asociados o referencias activas
    2. Se solicita confirmación antes de proceder con la eliminación
  3. Error de conexión con servicios de IA
    1. El sistema detecta fallas en APIs de OpenAI, Claude, Gemini o Grok
    2. Muestra mensaje específico del error y permite continuar en modo demo
    3. Registra el incidente para análisis posterior
  4. Timeout en generación de contenido
    1. Si el proceso excede 30 segundos, se muestra indicador de progreso
    2. Se proporciona opción para cancelar la operación
    3. Se preservan los datos ingresados para reintento posterior

REGLAS DE NEGOCIO

  1. El código del caso de uso debe ser único en el sistema
    1. Formato obligatorio: 2 letras + 3 números + nombre descriptivo
    2. Validación en tiempo real durante la captura
  2. El nombre del caso debe comenzar con verbo en infinitivo según normas ING
    1. Verbos válidos: gestionar, crear, consultar, administrar, configurar, procesar
    2. Validación automática mediante lista predefinida
  3. Solo usuarios con rol "Analista" o superior pueden crear casos de uso
    1. Verificación de permisos en cada operación CRUD
    2. Log de intentos de acceso no autorizado
  4. Se debe validar conectividad con servicios de IA antes de procesar
    1. Verificación de API keys válidas
    2. Test de conectividad con timeout de 5 segundos
  5. Registro automático en bitácora de todas las operaciones CRUD
    1. Timestamp, usuario, acción realizada y datos modificados
    2. Retención de logs por 12 meses para auditoría

REQUERIMIENTOS ESPECIALES

  1. Integración con múltiples APIs de inteligencia artificial
    1. OpenAI GPT-4: para generación de contenido general
    2. Claude Anthropic: para análisis de texto y mejoras
    3. Google Gemini: para procesamiento multimodal
    4. xAI Grok: para análisis contextual avanzado
  2. Generación de documentos DOCX con formato corporativo ING
    1. Fuente Segoe UI Semilight obligatoria
    2. Colores corporativos: azul ING (RGB 0,112,192)
    3. Listas multinivel con numeración específica (1-a-i)
  3. Tiempo de respuesta máximo 30 segundos para generación de contenido
    1. Indicador de progreso visual para operaciones largas
    2. Sistema de timeout con reintentos automáticos
  4. Validación en tiempo real según normas técnicas ING
    1. Verificación de formatos de código y nombres
    2. Sugerencias automáticas de mejora de contenido
  5. Sistema de caché para optimizar performance de AI Assist
    1. Cache local de respuestas frecuentes por 24 horas
    2. Invalidación automática cuando se actualizan prompts
