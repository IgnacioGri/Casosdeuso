Gestionar información del cliente

Nombre del Cliente: Banco Provincia
Nombre del Proyecto: Sistema de Gestión Integral
Código del Caso de Uso: BP001
Nombre del Caso de Uso: Gestionar información del cliente
Nombre del Archivo: BP001GestionarInformacionCliente

Descripción

Permite gestionar la información completa de los clientes del banco incluyendo consulta, actualización y seguimiento de la relación comercial.

Flujo Principal de Eventos

  1. Buscar datos de la entidad
    1. Filtros de búsqueda disponibles: DNI/CUIT, Apellido, Email, Número de Cliente
    2. Columnas del resultado de búsqueda: ID Cliente, Apellido y Nombres, Documento, Email, Estado
  2. Agregar una nueva entidad
    1. Campos de la entidad: clienteId (number, obligatorio), tipoDocumento (text, obligatorio), numeroDocumento (text, obligatorio)
    2. Al agregar se registra automáticamente la fecha y usuario de alta

Campos de la Entidad

  • clienteId: number(10) - Obligatorio
    Descripción: Identificador único del cliente
    Validación: Número entero positivo
  • tipoDocumento: text(10) - Obligatorio
    Descripción: Tipo de documento de identidad
    Validación: DNI, CUIT, CUIL
  • numeroDocumento: text(20) - Obligatorio
    Descripción: Número del documento de identidad
    Validación: Solo números, sin puntos ni guiones
  • apellido: text(100) - Obligatorio
    Descripción: Apellido del cliente
    Validación: Solo letras y espacios
  • nombres: text(100) - Obligatorio
    Descripción: Nombres del cliente
    Validación: Solo letras y espacios
  • email: text(150) - Opcional
    Descripción: Correo electrónico del cliente
    Validación: Formato email válido
  • fechaAlta: date - Obligatorio
    Descripción: Fecha de creación del registro
    Validación: Formato ISO 8601
  • usuarioAlta: text(50) - Obligatorio
    Descripción: Usuario que creó el registro
    Validación: Debe existir en el sistema

Reglas de Negocio

1. Solo usuarios autorizados pueden acceder 2. Se debe validar formato de documentos

Requerimientos Especiales

1. Integración con sistema legado 2. Logging de operaciones críticas

Precondiciones

  • El usuario debe estar autenticado en el sistema
  • El usuario debe tener permisos para gestionar la entidad

Postcondiciones

  • Los cambios se reflejan inmediatamente en la base de datos
  • Se genera una entrada en el log de auditoría
