Notas reunión - 12/02/2025 - Banco Macro - Módulo de Transferencias

Avanzamos con el diseño del nuevo módulo para transferencias dentro de la app/web. Participaron UX, tecnología, seguridad y negocio.

Primero se discutió lo técnico:

Todo debe auditarse, sin posibilidad de alteración.

AES-256 para cifrado de datos sensibles.

Integración con la API del BCRA para validar cuentas destino.

Sistema de detección en tiempo real para operaciones raras.

Backup de operaciones cada 5 minutos.

Tiene que funcionar bien con lectores de pantalla y ser accesible.

Después se habló mucho de la experiencia del usuario:

Hay que permitir buscar cuentas origen por CBU, alias o número de cuenta.

La carga de datos del receptor tiene que ser clara y guiada (alias, CBU, nombre, banco).

Si la cuenta está mal escrita o no existe, avisar en el momento.

Autenticación con token obligatorio para todo.

Si supera los $100.000, pedir segundo factor.

Errores comunes que hay que cubrir:

Saldo insuficiente → mostrar lo que tiene y sugerir el monto máximo.

Límite diario excedido → mostrar cuánto le queda y avisar que no puede seguir.

Si falla la conexión → guardar borrador automáticamente y permitir reintentar.

Si el banco destino rechaza → revertir la operación y avisar con motivo.

Token vencido → generar uno nuevo sin que tenga que volver atrás.

Si la cuenta origen está bloqueada → no permitir operar y mostrar mensaje claro.

Pantallas necesarias:

Buscador de cuentas con validación en tiempo real.

Formulario para carga de datos y cálculo automático de comisión.

Confirmación final antes de ejecutar.

Resultado de operaciones con filtros (monto, fecha, estado, banco, etc.).

Detalle de operación con estados (timeline).

Autenticación con token.

Programación de transferencias (se ejecutan 9:00 hs día elegido).

Campos clave de cada transferencia:

ID único (15 dígitos), CBU origen/destino, titular, monto, comisión, concepto (opcional), estado (pendiente, procesada, rechazada, cancelada), token, banco destino, timestamps, IP, dispositivo, usuarios de alta/modificación, motivo de rechazo si aplica.

Datos que se van a mostrar en resultados:

Número de operación, fecha/hora, últimas cifras de CBU origen y destino, nombre del receptor, monto, comisión, estado actual, banco destino.

Filtros útiles para buscar transferencias:

CBU o alias de cuenta origen, DNI/CUIT del receptor, fechas, estado, monto, banco receptor.

Límites y reglas de negocio:

Solo clientes con cuentas activas pueden operar.

El saldo tiene que alcanzar para cubrir el total (monto + comisión).

Límites por tipo de cuenta (estándar hasta $500.000, premium hasta $2.000.000 por día).

Si pasa cierto monto, se requiere doble autenticación.

Transferencias entre cuentas propias no pagan comisión; las otras, 0.5%.

Todo debe registrarse para cumplir con normativa del BCRA/UIF.

Las operaciones programadas se disparan solas a las 9 hs.

La sesión expira a los 10 minutos si no hay actividad.

Y para cerrar, los criterios de performance que hay que cumplir:

Cada validación debe tardar menos de 3 segundos.

Uptime mínimo: 99.9% durante horario bancario.

Compatibilidad con al menos 15 bancos locales.

Tiene que aguantar al menos 10.000 transacciones simultáneas.

Cumplir con los estándares PCI-DSS (datos sensibles, tarjetas, etc.).
