	PV001 Actualizar un proveedor 
 Definición del caso de uso

Objetivo	Agregar, actualizar y visualizar los proveedores
Si el usuario selecciono visualizar, los datos no se deben poder editar
Pendientes 	
Información adicional	


#	Acción
1.		El sistema muestra los siguientes datos para su edición 

	Dato	Clasificación	Referencia
	Datos Generales
	Razón Social	Obligatorio	Campo tipo input. Razón Social del proveedor. Ej: ARCOR S.A.I.C.
	Nombre de Fantasía	Obligatorio	Campo tipo input. Nombre de fantasía del proveedor. Ej: ARCOR
	Tipo de Identificador Impositivo	Obligatorio	Lista de selección única de tipos de identificador impositivo.
Si está agregando muestra solo los activos.
	Número de Identificador Impositivo	Obligatorio	Numérico de 11 números
	Condición de IVA	Obligatorio	Lista de selección única de condición de IVA.
Si está agregando muestra solo los activos.
	Tipo situación IIBB	Obligatorio	Lista de selección única de situaciones de IIBB.
Si está agregando muestra solo los activos.
	Numero Inscripción IIBB		Numérico
	Todos los medios de pago	Opcional	Check, por defecto está seleccionado
	Medio de pago	Obligatorio	Solo se visualiza si Todos los medios de pago = False 

Lista de medios de pago de proveedores activos que no sean Retención ordenados alfabeticamente.
 
	Extranjero	Opcional	Check, por defecto no está seleccionado
	Activo		Check, por defecto seleccionado
	Percepciones
	Percepción de ganancias		Check Box. Indica si se le percibe el impuesto al proveedor. Por defecto viene desmarcado
	Percepción de IVA		Check Box. Indica si se le percibe el impuesto al proveedor. Por defecto viene desmarcado
	Percepción de IIBB Capital		Check Box. Indica si se le percibe el impuesto al proveedor. Por defecto viene desmarcado
	Percepción de IIBB Provincia		Check Box. Indica si se le percibe el impuesto al proveedor. Por defecto viene marcado

2.		El sistema habilita la visualización de los botones “Guardar”, “Aplicar”, “Guardar y Agregar Nuevo”, “Cancelar”, de acuerdo a la seguridad del usuario identificado en el BackOffice.
3.		El usuario completa los datos. 
4.		El usuario selecciona la opción para aceptar los datos.
5.		El sistema verifica que:
a.	No existan dos proveedores con la misma razón social.
b.	Los campos obligatorios hayan sido completados.
6.		El sistema registra los datos, muestra un mensaje de grabación exitosa y regresa al CU que lo ejecutó.

#	Variación de las Acciones
1a	Si está agregando, el sistema traerá los campos vacíos.
4a	a.	El usuario selecciona la opción para cancelar la transacción.
b.	El sistema retorna al CU que lo llamo.
5a	Si los datos obligatorios no fueron ingresados, el sistema muestra un mensaje de advertencia.
6a	Si la registración no es exitosa, el sistema informa el motivo del impedimento.
